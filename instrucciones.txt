1. Contexto y Objetivo General del Proyecto

Crea un sistema de gestión web para el Instituto de Deportes del Estado Trujillo (INDET). El objetivo es centralizar la administración de la información, reservas y habitaciones, utilizando un diseño de interfaz superprofesional y moderno. A pesar de usar PHP puro, la interfaz debe ser visualmente impactante con animaciones fluidas, transiciones y efectos de parallax y 3D sutiles para una experiencia de usuario memorable.

2. Tecnologías y Estilo de Diseño

Lenguaje de Programación: PHP 8.x

Base de Datos: MySQL a través de phpMyAdmin.

Servidor Local: XAMPP.

Front-end: HTML5, CSS3 y JavaScript (Vanilla JS).

Animaciones y Efectos: Usa JavaScript para crear efectos de parallax en el fondo y animaciones visuales, como efectos 3D o de superposición al interactuar con los elementos.

Diseño: El diseño debe ser minimalista y elegante. La paleta de colores principal debe ser la de la bandera del estado Trujillo: verde (#006400), rojo (#FF0000) y blanco (#FFFFFF).

Elementos de la Interfaz:

Crea una 

cabecera moderna con enlaces a "Sobre La Institución", "Disponibilidad", "Contactos" y "Login".


El 

formulario de reserva debe estar visible en la página de inicio, con campos para "Fecha de llegada", "Fecha de salida" y "Habitación".


Incluye una sección con un título llamativo y una descripción, similar a la imagen proporcionada ("Somos la casa de los jóvenes").

Una galería de imágenes debe mostrar las instalaciones o actividades, con efectos visuales al pasar el cursor (por ejemplo, efectos 3D o de superposición).

3. Módulos y Funcionalidades Clave

El sistema se dividirá en los siguientes módulos:


Módulo de Información del INDET:

Muestra la información general del INDET: nombre, ubicación, contacto y los servicios ofrecidos.




Presenta imágenes de las instalaciones y los horarios de entrada y salida.


Incluye un formulario de contacto para enviar solicitudes de información y una sección de preguntas frecuentes.


Incorpora una sección para opiniones y comentarios.

Módulo de Habitaciones y Reservas:

Informa sobre los tipos de habitaciones disponibles, su capacidad y comodidades.


Permite a los usuarios seleccionar fechas y tipo de habitación para realizar una reserva.

Solicita los datos personales del huésped y confirma la reserva con un número de confirmación.


Módulo de Gestión de Habitaciones y Reportes:

Permite al personal gestionar el estado de cada habitación (ocupada, disponible, en limpieza).

El personal de mantenimiento puede asignar y gestionar las tareas de limpieza.


Genera reportes diarios del estado de las habitaciones y análisis del perfil de los clientes.


Muestra gráficas para evaluar el desempeño de las villas.

Módulo de Usuarios y Acceso:

Implementa un sistema de login seguro para empleados y clientes.

Permite el registro de nuevos usuarios con verificación de identidad.

Incluye un panel de administración para gestionar reservas , usuarios y asignar roles.



4. Estructura de la Base de Datos (Propuesta)

La IA debe crear un esquema de base de datos MySQL con las siguientes tablas:

users: Para almacenar la información de los usuarios (id, nombre, email, contraseña hasheada, rol).

rooms: Para gestionar los tipos de habitaciones (id, tipo, capacidad, descripción, precio, fotos).

reservations: Para guardar los detalles de cada reserva (id, id_usuario, id_habitacion, fecha_entrada, fecha_salida, estado).

room_status: Para el estado actual de cada habitación (id_habitacion, estado: 'disponible', 'ocupada', 'limpieza').

hotel_info: Para la información general del hotel y servicios.

reviews: Para almacenar los comentarios y calificaciones de los usuarios.
